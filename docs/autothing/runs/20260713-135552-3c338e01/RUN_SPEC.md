## What / why

Garrison keeps exactly three assumptions — a composition file, a Resolver that makes it real, and an orchestrator slot — and everything else is composed. This run rebuilds Garrison's middle around that sentence: **Duties** (work with a start and an end, provided by a Fitting, owning a skill) replace modes, task types, phases, and tiers; **Levels** (per-duty; each a leaf cell `{skill, target, effort}` or an ordered sequence of duty refs) replace tiers; the **Dispatcher** (itself a duty) replaces the tier classifier; **composition v4** absorbs duty definitions, levels, targets, provider references, and fitting config values (profiles die — a profile is just another composition); the **Muster page** becomes the one place the system is configured; the **Kanban board** becomes the duty observability surface. Plus the V2 carried items: daily-driver fixes, runtime expansion (openai-agents, garrison-call), the channel-Kanban bridge, voice on the phone, Quarters honesty.
