**E2 — Coordination stack.** The stack is two fittings. `coord-mcp` (Garrison-owned MIT stdio MCP, `fittings/seed/coord-mcp/scripts/server.mjs`) exposes exactly 7 tools: `begin_planning`, `end_planning`, `plan_heartbeat`, `plan_status`, `declare_intent`, `release_intents`, `coord_digest`. The planning lock is a dependency-free **file mutex** (`~/.garrison/coord/plan-locks/<repoSlug>.json`, 15-min TTL + heartbeat, atomic O_EXCL acquire), and intents are an append-only JSONL ledger (`~/.garrison/coord/intents/<repoSlug>.jsonl`). `coord-agentmail` runs the external `mcp_agent_mail` server (MIT + OpenAI/Anthropic rider, hard-isolated arm's-length in `~/.garrison/external`) providing leases (`file_reservation_paths`/`renew`/`release`), messaging (`send_message`/`reply_message`/`fetch_inbox`, polled inbox — no ack), build slots, and identities. **Beads is fully removed** (commit `7ddcaeb`) — no `coord-beads` fitting, no `bd` ever spawned; only vestigial dead comments/`consumes` remain. Neither coord fitting is in the default composition today, and the agent_mail clone is absent on this box.

**E13 — garrison-control.** It's a tool group inside the `mcp-gateway` fitting (not a standalone server): schemas in `gateway.mjs` `discoverTools()`/`dispatchTool()`, forwarders in `lib/tools.mjs`. Adding `record_improver_feedback` is a 3-edit pattern, and the tool layer **can write directly** to `~/.garrison` (it already reads it and resolves `GARRISON_HOME`) — natural target `~/.garrison/improver/feedback-queue.jsonl`. For the AskUserQuestion fallback: I verified against a real log that the answer **is** durably recorded in the session JSONL as a `tool_result` matched by `tool_use_id` plus a top-level `toolUseResult.answers` map — with the key gotcha that the answer line does not contain the literal string "AskUserQuestion" (match on the id, not a substring). The existing `jsonl-watcher.mjs` helpers give the exact path layout to reuse.
