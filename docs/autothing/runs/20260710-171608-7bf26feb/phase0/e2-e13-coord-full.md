FINDING-E2: end_planning(repo?) -> releases the lock, records the released plan for the next planner. Returns {status:"RELEASED"|"NOT-HELD"}.
FINDING-E2: plan_heartbeat(repo?) -> extends the holder's lock TTL (cannot resurrect an already-expired lock). Returns {ok, reason?}.

FINDING-E2: declare_intent(reason, area?, files?, repo?) -> appends a drift-signal intent row; required:["reason"]. Returns {status:"DECLARED", intent}.
FINDING-E2: release_intents(repo?) -> clears this session's intents. Returns {status:"RELEASED"}.
FINDING-E2: coord_digest(repo?, area?, files?) -> the same repo-scoped digest the hook injects {text, bytes, hasConflicts, conflicts, leaseConflicts}.
FINDING-E2: repo defaults to git-root of cwd (lib/repo.mjs repoRoot); session id = CLAUDE_SESSION_ID || COORD_SESSION || hostname-uuid8.

FINDING-E2: The planning lock is a dependency-free FILE mutex, NOT beads-backed (fittings/seed/coord-mcp/scripts/lib/plan-lock.mjs). Lock file: ~/.garrison/coord/plan-locks/<repoSlug>.json = {repo,session,summary,startedAt,heartbeatAt,expiresAt,ttlMs}. Default TTL 15min (COORD_PLAN_LOCK_TTL_MS / config lock_ttl_ms=900000). Atomic O_EXCL "wx" create for the acquire race; expired locks are taken over by acquireLock (never resurrected by heartbeat, never deleted by a stale holder's release). Waiters tracked in a sibling <repoSlug>.waiters.json (5-min freshness). forceReleaseLock is the admin override (surfaced as `coord release-lock`).
FINDING-E2: Intents are an append-only JSONL ledger at ~/.garrison/coord/intents/<repoSlug>.jsonl (lib/intent-store.mjs), rows {repo,session,area,files,reason,ts}. No expiry beyond the lookback filter; removeIntentsBySession rewrites the file.
FINDING-E2: The digest (lib/digest.mjs, MAX 1200 bytes ~ few hundred tokens) surfaces (a) planning-lock-held-by-other, (b) intent conflicts + awareness, (c) agent_mail file-lease conflicts + awareness, plus a standing NUDGE telling the agent to call begin_planning before substantial work.
